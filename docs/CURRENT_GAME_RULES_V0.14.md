# CardEater 当前游戏规则汇总（v0.14）

> 用途：供其他 AI、设计师或后续开发者评估当前游戏。本文只记录代码在 `d3ff13d` / `0.14.0` 时的实际规则，不包含尚未确认的重设计提案。
>
> 更新时间：2026-07-21。若本文与代码冲突，以 `js/config.js`、`js/data.js`、`js/rules.js`、`js/quests.js`、`js/items.js`、`js/engine.js` 和 `js/shop.js` 为准。

## 1. 游戏定位与目标

CardEater 是一个原生 H5 单局构筑肉鸽。玩家逐张处理牌：

- 向下或点击“吃掉”执行【吃】。
- 向上或点击“弃掉”执行【弃】。
- 每张牌分别拥有吃分、弃分和至多一个主要效果。
- 可食用牌通常适合吃，不可食用牌通常适合弃，但卡牌效果、规则、道具和任务会鼓励反向选择。

一局共 15 轮，分数跨轮累计。只有以下轮次检查失败条件：

| 检查轮 | 累计分数目标 |
| ---: | ---: |
| 5 | 120 |
| 10 | 750 |
| 15 | 4000 |

- 第 5 或第 10 轮结算后累计分数不足，立即失败，不能进入商店。
- 第 15 轮达到 4000 分即通关，否则失败。
- 其他轮次没有最低分要求。
- 总分与单次运算被限制在 `±9e15`，防止出现 `Infinity` 或 `NaN`。

## 2. 每轮实际流程

普通轮次：

1. 从符合条件且本局未选过的规则中随机出现 3 条。
2. 玩家必须选择 1 条；该规则永久加入本局。
3. 洗混完整永久牌组。
4. 按餐盘上限抽取本轮登场牌。
5. 玩家按固定牌序逐张选择吃或弃。
6. 牌堆为空时，如可以重洗，可选择重洗或结束本轮。
7. 结算牌分、规则、道具、任务、金币和累计分数。
8. 未失败时进入商店。
9. 玩家可买牌、买道具、扩容、删牌或刷新，然后主动进入下一轮。

第 3、6、9、12 轮在“选择规则”之后额外执行一次危险任务三选一，玩家不能跳过任务。

任务成功后的道具不会在当前轮立刻生效，而是在下一轮选择规则前装备。

## 4. 当前关键字定义

| 关键字 | 代码中的定义 |
| --- | --- |
| 摧毁 | 将指定实体牌从本局永久牌组移除；本轮之后也不会再次抽到。所有同义描述统一为“摧毁”。 |
| 相邻 | 读取初始牌序中紧挨本牌的前一张、后一张或两侧牌。部分相邻效果也读取上一张实际行动。 |
| 成长 | 永久提高这张实体牌写明的吃分或弃分；重洗和后续轮次保留。 |
| 生成 | 向永久牌组加入新实体牌；达到牌组或该效果的生成上限时失败。 |
| 重洗 | 将本轮已处理且仍存在于永久牌组的牌重新洗回抽牌堆。 |
| 净化 | 移除尚未结算的负面后续效果；已扣分数不返还。 |
| 追溯 | 读取本轮此前的行动或得分，不会重新触发此前卡牌。 |
| 蓄势 | 为之后符合条件的若干张牌保存加成；不符合条件的牌不消耗次数。 |
| 位置 | 检查本牌在本轮行动顺序中的首位、次位、中位或末位。 |
| 规模 | 按永久牌组张数、类别数或不同卡名数计算。 |
| 复制 | 复制指定分数数值，不复制被复制牌的效果。 |
| 经济 | 改变金币、卡价、刷新费用或扩容费用。 |
| 储存 | 把数值保存在某张实体牌上，跨轮保留，直到效果写明的方式兑现并清空。 |
| 预判 | 读取本轮尚未处理的牌，不改变牌序，也不触发那些牌。 |

## 5. 餐盘、牌序与候补区

- 每轮先洗混完整永久牌组。
- 本轮登场数为 `min(牌组张数, 餐盘上限)`。
- 超出餐盘上限的牌成为“未登场牌/候补牌”，本轮不会被普通操作处理。
- 候补牌不是永久移出；下一轮重新与全部牌一起洗混。
- 系统记录候补总数及各类别数量，供卡牌和道具读取。
- 玩家正常情况下只能处理当前牌堆顶端的一张牌，不能自由选择牌、换位或调整顺序。
- 相邻、下一张、首位和末位等效果由随机后的固定顺序以及实际行动历史判断。

餐盘上限是当前版本唯一的系统级大牌组限制。当前明确不使用：

- 构筑密度倍率。
- 牌组携带费。
- 大牌组额外卡价。
- 吃牌金币饱腹上限。
- 删牌返还或删牌奖励。

## 6. 吃、弃、计分与金币

### 6.1 单张行动分数

当前单张牌大致按以下顺序结算：

```text
选择侧的牌面分
+ 已选永久规则的固定加分
+ 已有蓄势的固定加分
+ 道具的即时固定加分
+ 当前任务的行动惩罚
→ 乘以作用于该张牌的蓄势倍率
+ 该牌的即时效果加分
= 本张最终得分
```

- 当前牌产生的新蓄势通常只影响后续牌，不影响自己。
- 部分效果可以改用另一侧分数或吃/弃两侧中的较高分。
- “负分吃牌”等条件读取该张牌结算后的最终得分，不只看印刷数值。

### 6.2 本轮与累计分数

```text
牌面与效果分 = 本轮所有行动牌最终得分之和
最终倍率 = 卡牌倍率 × 道具倍率 × 任务永久倍率 × 所有已达成规则倍率
本轮得分 = 牌面与效果分 × 最终倍率
累计分数 += 本轮得分
```

- 所有倍率彼此相乘，不是相加。
- 每一条已选永久规则每轮都会重新检查。
- 固定加分规则直接作用到单张牌，不再作为末尾倍率出现。
- 结算栏会列出各类别分数及每个生效倍率。

### 6.3 金币

- 每轮每张不同实体牌第一次被吃时，基础金币 +1。
- 同一实体牌经重洗后再次吃，不重复获得基础金币。
- 弃牌默认不产生金币，必须依靠卡牌或道具效果。
- 卡牌、道具产生的额外金币在轮末加入。
- 金币跨轮保留，用于购买、刷新、扩容和删牌。

## 7. 重洗规则

- 重洗不是基础行动，必须由卡牌或道具提供次数。
- 永久牌组必须不超过 10 张才能重洗。
- 本轮至少处理过一张仍存在于永久牌组的实体牌后，才有可重洗对象。
- 被摧毁的牌不会被重洗回来。
- 使用 1 次重洗会消耗 1 次重洗次数，把已处理牌随机洗回当前抽牌堆。
- 多个重洗来源直接相加，可以在同一轮使用多次。
- 重洗后的牌可以再次触发普通牌面效果和永久成长。
- 标记为每轮一次的效果不会因重洗再次触发。
- 重洗次数不会跨轮保留。
- 当前规则池没有任何“重洗后获得倍率”或“重洗次数对应倍率”的规则。
- 单轮最多执行 400 次行动，达到上限后强制清空剩余牌。

当前直接重洗来源：

- 任务道具“重启按钮”：10 张以内每轮 +1 次。
- 商店道具“回卷发条”：10 张以内每轮 +1 次。
- 卡牌“袖珍洗牌机”：弃置并摧毁自身；摧毁后牌组不超过 10 张则本轮 +1 次。
- 卡牌“秒表”：10 张以内每轮首次弃置时本轮 +1 次。

## 8. 永久规则系统

### 8.1 抽取与叠加

- 每轮必须从 3 条规则中选择 1 条。
- 已选择规则永久保留到本局结束，且不会再次出现。
- 规则有最早解锁轮次。
- 部分类别、牌组尺寸和关键字规则会检查当前永久牌组是否具备条件，再决定能否进入候选池。
- 资格检查读取完整永久牌组，不读取本轮实际登上餐盘的牌。
- 草案会尽量避免 3 条候选都属于同一大类，但不保证每条都容易完成。
- 固定加分规则每张符合牌都生效；条件倍率规则在轮末达成时生效。
- 一局最多获得 15 条永久规则。

## 9. 危险任务

### 9.1 通用规则

- 任务只在第 3、6、9、12 轮出现。
- 每次从符合解锁条件且本局未出现过的任务中随机提供 3 个，玩家必须选 1 个。
- 基础任务分数目标分别为 25、90、300、1000；部分任务再乘自己的系数。
- 金币清零和永久虚空牌在选中任务时立即生效。
- 行动扣分在当轮处理卡牌时生效。

## 12. 商店与经济规则

### 12.1 商品

- 每间商店提供 3 张卡和 3 件道具。
- 买走一个货位后不会自动补货。
- 达到单卡持有上限的卡会在生成商品前被过滤。
- 永久牌组上限为 160 张。
- 卡牌折扣叠加后最低价格为 1。
- 商店道具只能持有一件同名道具。

### 12.2 刷新

- 每进入一间新商店，付费刷新从 1 金币开始。
- 同一商店内刷新费用依次为 1、2、3、4……。
- 刷新会同时更换卡牌和商店道具。
- 免费刷新费用为 0，但仍会推进刷新计数；例如两次免费刷新后，下一次付费刷新是 3 金币。
- 合盖餐罩可在全部牌登盘时让付费刷新 -1，但最低仍为 1。

### 12.3 餐盘扩容

- 每次扩容永久增加 1 个餐盘位置。
- 基础价格公式：`2 + n(n+1)/2`，`n` 为此前扩容次数。
- 价格序列：2、3、5、8、12、17、23……。
- 餐盘量尺使每次扩容 -1，最低 1。
- 餐盘硬上限为 160。

### 12.4 删牌

- 第一次删牌免费，之后价格依次为 3、6、9、12……。
- 删牌不返还金币，也不提供任何系统奖励。
- 牌组至少保留 1 张牌。
- 删除的是指定实体牌，永久成长和储存状态随该实体一起消失。
- UI 必须二次确认，防止误触。
- 当前删牌的唯一收益是提高目标牌进入餐盘的概率和减少负面牌。

## 13. 当前构筑路线（代码已有支持，不代表平衡良好）

1. 吃牌与经济：通过吃不同实体牌获得基础金币，再用可食用类别规则和经济牌扩大收益。
2. 纯弃/少吃：依赖不可食用牌弃分、弃牌倍率、弃牌金币道具及反食性奖励。
3. 小牌组重洗：保持 10 张以内，通过重洗重复计分、效果和成长；重复吃不重复获得基础金币。
4. 大牌组保底：依靠更多牌、类别覆盖、规模卡和候补条件；受餐盘上限限制本轮登场数量。
5. 位置/相邻：利用首位、末位、前后相邻、上一行动和下一张牌条件。
6. 摧毁/成长：摧毁自己或其他实体牌，换取成长、金币、生成和倍率。
7. 生成：永久扩大牌组并利用生成牌专属加分、金币和任务。
8. 负分牺牲：先主动承受负分，再让后续卡牌获得加分或倍率。
9. 类别专精/多样性：集中某一类别，或同时处理多种类别获得奖励。

## 14. 交给其他 AI 时应同时说明的已知问题与边界

以下不是新规则，而是当前版本需要被评估的问题：

- 作者实测约 20 局后，认为整体缺少明确驱动、局内成长感和“再来一局”的欲望。
- 每局开局牌组固定，每轮流程高度一致。
- 当前餐盘只限制登场数量，玩家不能主动调整随机牌序；因此部分位置和相邻效果更接近随机命中。
- 规则每轮永久增加，后期可能形成大量条件和乘区叠加。
- 当前卡牌名义效果类型很多，但实际体验中仍感觉重复、描述负担大、部分组合接近白板。
- 当前真人体验认为大牌组仍容易同时获得更高经济与更高基础分；模拟胜率接近不等同于有趣。
- 不希望用局外永久数值成长解决重复游玩问题。
- 不希望把游戏改造成做菜、餐厅、顾客、订单或评审主题；“餐盘”只是机制接口。
- 已确认的硬约束：虚空牌吃 -2、弃 -1、无效果、无类别；删牌不返还；大牌组的系统级约束目前只使用餐盘上限；重洗不再绑定最终倍率规则。
- 在新的核心方向得到作者确认前，不应直接重写现有玩法代码。
