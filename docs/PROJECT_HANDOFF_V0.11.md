# CardEater v0.11 项目交接

更新时间：2026-07-19

## 一分钟项目概览

- 项目是无框架、无运行时依赖的原生 HTML/CSS/ES Modules 静态网页，GitHub 仓库为 `TheO1dF/card-eater-game`，Cloudflare Pages 地址为 `card-eater-game.pages.dev`。
- 当前版本 `0.11.0`，状态结构 `schema_version: 8`；一局 15 轮，第 5 / 10 / 15 轮检查累计 100 / 500 / 1800 分。
- 内容规模：110 张卡牌、64 种底层效果、76 条永久规则、12 种危险任务、12 件任务道具、16 件商店道具。
- 核心操作只有吃与弃；局外决策集中在买牌、永久餐盘扩容和付费删牌之间。
- 唯一的大牌组系统限制是永久餐盘。默认容量 10，每轮从洗好的永久牌组中随机截取最多容量张牌登场；不要重新添加构筑密度、携带费、金币饱腹或牌组税。

## v0.11 解决的问题

### 重洗可用且可叠加

旧版袖珍洗牌机先检查 11 张牌组再摧毁自己，导致 10 张牌组买入后永远判定失败；最后一张牌处理完又会直接结算，玩家没有按重洗的机会。

现在：

1. `js/reshuffle.js` 集中提供 `getReplayableCards`、`getReshuffleStatus` 和 `activateReshuffle`。
2. 袖珍洗牌机按“摧毁自身后的牌组张数”检查 10 张上限。
3. 所有来源直接累加 `reshuffle_charges`，每次重洗只消耗 1 点，不设单次上限。
4. 空牌堆仍可重洗时不会自动结算；底部显示可洗回张数、剩余次数，并出现“不重洗 · 结束本轮”。
5. 重洗按钮有可用脉冲，卡堆有左右交错的像素步进动画，提示会写明洗回张数。
6. 新增罕见卡“秒表”：10 张内每轮首次弃掉获得 1 次重洗；新增售价 5 的商店道具“回卷发条”：10 张内每轮获得 1 次。
7. 新增“回锅加热”和“二次盛宴”两条规则，分别检查重洗后行动数和重洗后得分。

### 前期经济与非位置维度

将三张低命中率位置/类别牌改为不同资源轴，而不是只换数值：

- 橙子：按永久牌组水果数量产生结算金币，形成水果经济核心。
- 热狗：每轮首次吃掉使随后商店卡价 -1，形成吃牌商店经济。
- 卫星：按本轮未登场牌数量产生结算金币，直接读取餐盘余量。
- 菠菜：改为吃掉后强化本轮后续所有蔬菜，不再要求固定第 2 位。
- 商店新增售价 4 的“零钱餐巾”，每弃 2 张获得 1 金币，每轮最多 3 次，给纯弃路线早期经济入口。

所有移除永久牌的语义仍统一为【摧毁】；新效果分别归入【重洗】【经济】【规模】【蓄势】。

### 长线投资提前回报

- 金苹果从稀有降为罕见，仍是每次吃掉永久 +1。
- 发芽种子起始吃分从 0 提至 1，重洗后每次吃永久 +2。
- 豆芽从累计吃 3 次成长改为 2 次。
- 糖果每次储存从 +2 提至 +3，两次即可到 6 分上限。
- 魔法帽降为 4 金币并可第 1 轮出现；育苗盘、年轮尺降至 5；餐盘量尺降至 4；夜市会员卡降至 6 且第 2 轮可出现。
- 任务基准从 30 / 120 / 500 / 2200 改为 25 / 90 / 300 / 1000。

### 商店三选一成本

- 永久扩容价格：2、3、5、8、12、17、23……，公式为 `2 + n(n+1)/2`，每次容量 +1。
- 删牌价格：0、3、6、9……；删牌永不返还金币，唯一收益是提高目标牌登场概率。
- 刷新仍是 1、2、3、4……；免费刷新仍推进次数，刷新后商品可以正常购买。
- 牌组安全上限仍为 160；商店会过滤达到持有上限的卡，避免“金币充足却不能购买”。

## 15 轮验证，不再只是“不崩溃”

`scripts/simulate-run.mjs` 是可重复的真实策略回归：不赠送金币、不赠送全道具，直接调用正式游戏的规则抽取、任务、卡牌引擎、重洗、商店、扩容、删牌和里程碑服务。

```powershell
npm run simulate
```

默认输出：

- 种子 1、平衡策略的逐轮日志；该局完整通过 15 轮，包含实际买牌和多次扩容。
- 平衡、小牌组、大牌组三种策略各 40 个固定种子。
- 当前基线通关率约为 62.5% / 57.5% / 72.5%。这是回归下限，不代表真人最优策略；测试只要求三种策略的 20 种子通关率均不低于 45%。
- 小牌组种子 6 会完整通关并多次重洗，专门覆盖叠加路径。

旧的 15 轮测试会预装全部 26 件道具，只能证明状态不产生 NaN。它保留为安全上限测试，但不能再作为平衡结论；新的策略测试才是里程碑调整依据。

## 开发与部署

```powershell
npm test
npm run simulate
npm run check
git push origin main
```

`npm run check` 会做语法检查、全部测试和静态构建，产物进入 `dist/`。Cloudflare Pages 应继续连接 GitHub `main`；不要手工修改 `dist/` 后期待源码同步。

本轮尝试连接可控制浏览器做视觉点击测试，但当前会话返回的浏览器列表为空，因此没有声称完成可视化浏览器复验。DOM、CSS、交互状态由单元/集成测试和构建检查覆盖；下一位 AI 若具备浏览器实例，应优先复验：空牌堆重洗提示、结束按钮布局、连续两次重洗动画，以及商店 2/3/5 扩容价格。

## 下一轮优先观察

1. 真人是否理解“重洗次数可叠加”，是否会在空牌堆主动选择结束。
2. 大牌组 40 种子通关率仍高于另外两类；先观察真人数据，只通过卡牌/规则权重微调，不新增第二条系统税。
3. 卫星只提供最多 3 金币，防止未扩容大牌组重新变成线性经济最优；若偏强先降出现权重或上限。
4. 2/3/5 的前期扩容明显变轻，但 12/17 后仍会逼迫玩家在买卡、扩容、删卡之间取舍。
5. 真机浏览器测试需补上 390×844 移动端和 1366×768 桌面端截图；当前没有可控制浏览器实例。
