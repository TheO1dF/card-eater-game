## v0.14 完成内容

### 收银台与刷新

收银台 `U007` 仍是普通卡、买价 3 金币、弃分 -2，并会摧毁自身；奖励从 1 次改为 2 次免费刷新。免费刷新仍推进刷新次数，因此两次免费分别抵掉原本 1 与 2 金币的刷新，第三次刷新显示 3 金币。

对应回归测试会真实调用两次免费刷新，并检查第三次价格。

### 移除所有重洗倍率规则

以下六条规则已经从 `RULE_LIBRARY` 删除：

- 再来一遍、循环过载。
- 熟能生巧、永动节拍。
- 回锅加热、二次盛宴。

永久规则从 76 条变为 70 条，规则池中不存在 `requires_reshuffle`，也不存在 `min_reshuffles`、`repeat_card_actions`、`post_reshuffle_actions`、`post_reshuffle_score` 四种作用域。引擎保留相应解析器只是为了兼容历史数据；新对局不会抽到这些规则。

重洗本身仍可叠加，仍会重复处理牌、触发非每轮一次效果和永久成长，但不再额外打开规则倍率乘区。为保持流派胜率，小牌组尺寸规则调整为 10 张 `×1.25`、8 张 `×1.75`、7 张 `×2`；大牌组 12/16 张规则降为 `×1.02 / ×1.05`，任务道具无底封条降为 `×1.05`。

### 经济道具回报周期与差异化

`js/shop.js` 现在按 `max_shop_round` 过滤投资道具。后期不会再出现已经无法回本的金币、刷新、扩容、返现或折扣道具。商店卡片会显示“第 X–Y 轮”。

被覆盖的道具已经拆开：

- 旧罗盘不再是首张 +1，而是“第一张正确分类后，下一张 +2”，因此不再被盘沿夹完爆。
- 候补餐罩改为购卡返现。
- 保温软垫改为“至少 4 张未登盘时最终分 ×1.08”，不再与候补餐罩互为首末位镜像。
- 拆信刀改为每轮首次【摧毁】牌 +3，和每张【相邻】牌 +1 的铰链夹采用不同频率。
- 牛皮纸袋改为“硬吃不可食用牌后，下一张弃牌 +2”，不再和缺口餐叉使用同一个反食性即时加分模板。

### 随时查看规则与道具

固定顶栏现在为“组 / 规 / 具 / 任 / 声音”：

- “规”打开本局永久规则列表，显示名称、条件、倍率和选择顺序。
- “具”打开本局已获得道具，显示图标、来源类型、角色和完整效果。
- 两个入口在游玩、规则选择、结算和商店阶段都可用；空列表也会解释获得方式。
- 手机隐藏阶段文字，为五个按钮保留空间；Edge 实测无重叠。

### 图像与商店布局

- 卡牌卡图区域移除白色盘面，恢复为中性米色像素方格；实际牌堆下方的白色餐盘仍保留。
- 罕见卡整张纸张从偏绿 `#d8e7c2` 改为中性米色 `#e8ddc4`。
- `scripts/optimize-sprites.mjs` 会把每格近黑背景转为透明，按连通主体去除跨格碎片，再缩放到 78% 安全区并居中。
- 运行时仍使用 `assets/shop-items-atlas-v013.webp`，但缓存版本已升到 `v=14`。Edge 逐格测量 24 个图标，最大中心偏差为 1px。
- 桌面商店从 940px 缩到 820px；商品卡、道具卡、区段间距和牌组区高度均压缩。手机仍为单列滚动布局。

## 后续注意

1. 不要重新加入重洗倍率规则；如小牌组偏弱，调尺寸规则、卡牌或道具，不把倍率绑回重洗次数。
2. 新经济道具必须同时给出价格、最早轮次和最晚轮次，并按剩余可触发次数估算回本。
3. 新道具需检查是否被更便宜、更宽松的道具严格覆盖；镜像效果也要有不同频率、资源或风险。
4. 图集导出必须使用质量 `0.82`，否则会重编码所有旧 WebP。
5. 每次 UI 改动继续按 `AGENTS.md` 使用 Edge 双视口冒烟，不依赖 Chrome 扩展连接。
