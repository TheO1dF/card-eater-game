# CardEater v0.12 项目交接

更新时间：2026-07-19

## 一分钟项目概览

- 原生 HTML/CSS/ES Modules 静态网页，无前端框架与运行时依赖；GitHub 为 `TheO1dF/card-eater-game`，Cloudflare Pages 为 `card-eater-game.pages.dev`。
- 当前版本 `0.12.0`，状态结构 `schema_version: 9`；一局 15 轮，第 5 / 10 / 15 轮检查累计 100 / 500 / 1800 分。
- 内容规模：110 张卡牌、76 条永久规则、12 种危险任务、12 件任务道具、24 件商店道具。
- 唯一的大牌组系统约束仍是永久餐盘：默认每轮最多登场 10 张，商店可永久扩容。不要重新引入构筑密度、携带费、金币饱腹、牌组税或删牌返还。
- 核心局外选择仍是买牌、扩容、删牌；删牌价格 0 / 3 / 6 / 9……且没有返还。

## v0.12 完成的五件事

### 1. 目标和规则说明进入主流程

欢迎页现在直接展示：

- 明确目标：第 5 / 10 / 15 轮累计 100 / 500 / 1800 分，目标轮未达成立即结束。
- 三步循环：选永久规则 → 吃/弃 → 商店买牌/扩容/删牌。
- 吃与弃的基础方向。

每轮规则选择页还会动态显示当前累计分、距离目标差值、剩余轮数、进度条、当前永久规则数量。规则卡标注“基础 / 进阶 / 后期”和最早解锁轮次。

### 2. 强规则采用手工分层

`js/rules.js` 导出 `getRuleUnlockRound(rule)`，抽取与 UI 使用同一最早轮次。容易稳定完成的高倍率规则不再首轮出现，例如：

- 完美分类、无伤清台、营养均衡：第 3 轮。
- 四分饱、顺手整理、苦尽甘来、三类别、吃弃四拍：第 2 轮。
- 袖珍牌组 ×1.85：第 4 轮。
- 九类“类别节拍”规则：第 2 轮。

这是逐条设计，不是“倍率超过某值自动锁定”。需要牺牲、组件、关键字或复杂行动序列的规则仍可用较高回报匹配难度。

### 3. 重洗保留爽感，但不再重复印钱

同一 `card_uuid` 每轮只有第一次被吃会产生 1 基础金币。重洗仍能：

- 再次触发牌面效果。
- 再次触发不受每轮一次限制的永久成长。
- 增加行动数并满足重洗/重复行动规则。
- 叠加多个来源的重洗次数。

重洗专属倍率也已下调并延后：“再来一遍”×1.25（第4轮）、“循环过载”×1.5（第7轮）、“熟能生巧”×1.35（第5轮）、“永动节拍”×1.7（第8轮）、“回锅加热”×1.35（第5轮）、“二次盛宴”×1.5（第7轮）。

结算栏会同时显示吃牌动作次数、实际计金实体牌数和基础金币，避免玩家误以为是 bug。

### 4. 商店道具从 16 件扩到 24 件

新增 8 件售价 2～4 的基础道具：缺口餐叉、牛皮纸袋、三格调味盒、盘沿夹、保温软垫、分类标签、尾单夹、换色弹簧。它们分别读取首次行动、精确位次、首末位、餐盘留牌、连续正确分类等不同状态，不是单纯数值换皮。

旧核心道具重新定价和分期：

- 回卷发条：9 金币，第 4 轮起。
- 夜市会员卡：10 金币，第 5 轮起。
- 魔法帽：8 金币，第 4 轮起。
- 零钱餐巾：7 金币，第 3 轮起。
- 餐盘量尺：6 金币，第 3 轮起。

商店 UI 会按售价与解锁轮次标出基础 / 进阶 / 核心道具。

### 5. 删牌必须二次确认

商店点击永久牌组中的卡不会立即删除，而是打开 `role="alertdialog"` 的确认浮窗。浮窗展示牌图、名称、类别、食性、吃弃分、完整效果、当前删牌价格和不可撤销说明；取消不改变牌组，确认后才调用交易服务。点击遮罩、按 Escape 或取消按钮均可关闭。

## 验证与浏览器约定

本仓库以后统一使用 Microsoft Edge 做 UI 冒烟测试；根目录 `AGENTS.md` 固化了此约定。流程：

```powershell
npm run check
npm run simulate
```

随后启动本地静态服务和无头 Edge，以 DevTools 端口运行 `scripts/browser-smoke.mjs`。测试必须覆盖 390×844 移动端与 1280×800 桌面端，截图和 JSON 保存到 `.artifacts/smoke-v012/`。重点断言：

- 欢迎页目标与三步流程可见且无横向溢出。
- 规则页目标、差值、剩余轮次、进度条和分层标签存在。
- 牌组随时查看、关键字和容量信息不冲突。
- 删牌点击只开确认；取消牌数不变；确认牌数恰好 -1。
- 连续推进至第 4 轮，任务、商店刷新和刷新后购买正常。
- 控制台无页面脚本错误，移动端无横向溢出。

v0.12 实测结果：`npm run check` 的 68 项测试、语法检查和构建全部通过；`npm run simulate` 的平衡 / 小牌组 / 大牌组 40 种子通关率为 57.5% / 55% / 72.5%。Edge 两种尺寸均推进至第 4 轮，控制台错误与硬断言失败均为 0；删牌取消保持 7 张、确认后变为 6 张，刷新后在金币充足时可以正常购买。完整报告位于 `.artifacts/smoke-v012/report.json`（该目录默认不提交 Git）。

## 下一轮优先观察

1. 真人是否能从开局页立即复述三个累计目标，以及是否理解规则是永久、条件未达成不扣分。
2. 去除重洗重复金币后，小牌组经济是否过紧；优先调整专属经济牌或基础道具，不恢复重复金币。
3. 24 件道具池中，8 件基础道具是否确实成为前期可买选择；核心道具不要再次同时降价和提前。
4. 强规则的最早轮次应继续逐条评估，不要新增全局倍率阈值。
5. 大牌组仍只用餐盘约束；若强度偏高，先调卡牌/规则/商店权重。
