# CardEater v0.13 项目交接

更新时间：2026-07-19

## 一分钟概览

- 原生 HTML / CSS / ES Modules 静态网页，无框架与运行时依赖；GitHub：`TheO1dF/card-eater-game`，Cloudflare Pages：`card-eater-game.pages.dev`。
- 当前版本 `0.13.0`，存档结构 `schema_version: 10`；一局 15 轮，第 5 / 10 / 15 轮检查累计 `120 / 750 / 4000` 分。
- 内容规模：110 张卡、76 条永久规则、12 个危险任务、12 件任务道具、24 件商店道具。
- 唯一的大牌组系统约束仍是永久餐盘：初始 10 格，商店可永久扩容。不要重新加入构筑密度、携带费、金币饱腹、牌组税或删牌返还。
- 玩家核心商店决策是买牌、扩容、删牌；删牌费用 `0 / 3 / 6 / 9…`，不会返还金币，且必须二次确认。
- UI 浏览器测试统一使用 Microsoft Edge，约定写在根目录 `AGENTS.md`。

## v0.13 为什么这样平衡

v0.12 的 40 种子通关率为平衡 57.5%、小牌组 55%、大牌组 72.5%。问题并不是单张规模牌，而是大牌组在前期从 7 张自然买到 10 张时，不用扩容就能多获得行动、基础分与吃牌金币；中后期又能叠加规模封顶和大小规则。

本版没有增加新的牌组惩罚，而是只调整现有内容层：

1. 大牌组规则降到 12 张 `×1.05`、16 张 `×1.1`；任务道具“无底封条”降到 `×1.1`。
2. 兔子、丰收篮、披萨、卫星、星图、兔群头领、队长的规模封顶下调，让大牌组保留基本分保障，但不能同时获得最高连击。
3. 小牌组尺寸规则为 10 张 `×1.17`、8 张 `×1.6`、7 张 `×1.8`。尺寸规则只有当前牌组已满足时才进入候选；小牌组规则抽取权重为普通规则的 `1.25`，不是必出。
4. “随身盐盒”成为前期小牌组启动器：牌组不超过 8 张时，每轮首次吃和首次弃各 `+2`。
5. 模拟器的小牌组策略会回避无计划生成、主动维持 8 张并在经济允许时支付删牌费；这让测试比较的是合理构筑，而不是故意失控的牌组。

调整后 `npm run simulate` 的固定 40 种子通关率为：平衡 27.5%、小牌组 22.5%、大牌组 30%，最大差距 7.5 个百分点。目标从 1800 提到 4000 后仍有稳定通关样本；种子 1 的平衡构筑为 6701 分，种子 10 的小牌组能多次重洗并通关。81K 属于真人极限成型组合，不适合直接作为中位门槛。

## 任务与虚空牌

- 虚空牌 `Q001` 固定为吃 `-2`、弃 `-1`、无效果、无类别。吃侧更差，因为吃仍会产生基础金币。
- 任务只在第 3 / 6 / 9 / 12 轮出现；要求、当前惩罚和奖励可随时查看。
- 完成任务后只把奖励排入待生效队列，结算栏写明“任务完成，下一轮生效”，下一轮开始才装备。
- 商店道具不能无条件强过任务道具；强组件通过价格与解锁轮次限制，而不是削成无用白板。

## 不要回退的约束与下一轮观察

1. 不要用基础金币上限或牌组税平衡大牌组；先调规模封顶、规则倍率、商店道具和卡池权重。
2. 不要让删牌返钱；删牌价值只来自提高核心牌进入餐盘的概率。
3. 不要把回卷发条重新降到早期低价。重洗能叠加且可重复成长，本身是高价值资源。
4. 新道具避免固定第 N 张、中间位和同模板换数值；优先使用未登场牌、唯一名称/类别、牌面选择、来源、关键字和商店资源等不同维度。
5. 后续真人数据应记录每个检查点的死亡率、牌组/餐盘大小、扩容次数、删牌次数、重洗次数、道具购买和最终分布。极限分不等于合理目标，目标优先看分位数与流派胜率差。
6. 改 UI 后必须继续用 Edge 跑手机与桌面冒烟；仅通过单元测试不算完成。
