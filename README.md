# 吃牌肉鸽 · CardEater Classic

一个无依赖、移动端优先的原生 H5 卡牌构筑肉鸽。玩家逐张处理餐盘中的牌：向下吃、向上弃，或左右侧滑【后置】当前牌，通过软控制顺序完成连击、成长与资源转换。

Fable5 评审前的英文规则与设计材料见 [`docs/CARDEATER_ENGLISH_RULES_AND_FABLE5_BRIEF.md`](./docs/CARDEATER_ENGLISH_RULES_AND_FABLE5_BRIEF.md)；89 张卡的统一重绘提示词与精确切图表见 [`docs/CARD_ART_SPRITESHEET_PROMPTS_V0.17.md`](./docs/CARD_ART_SPRITESHEET_PROMPTS_V0.17.md)。

当前 89 张卡牌已经全部应用统一的 v0.17 新卡图。资源目录与重新切图方式见 [`assets/README.md`](./assets/README.md)。

## 本地运行

ES Modules 需要通过 HTTP 打开，不能直接双击 `index.html`：

```powershell
npm start
```

访问 `http://localhost:8080`。验证命令：

```powershell
npm test
npm run check
npm run simulate
```

`npm run build` 会生成可直接部署到 Cloudflare Pages 的 `dist/`。

## v0.18 核心玩法

- 当前活跃卡池为 89 张、8 个类别，卡池不是固定上限，后续可以按流派逐步扩充；六张基础教学牌只保留吃弃点数与风味文字。
- 每张实体牌每轮最多后置一次，标记会跨自动重洗保留；合约只奖励真正触发的后置牌效，不奖励空转侧滑。
- 初始 7 张全部不同：4 张可食用、3 张不可食用。两张水果都带【水果连击】，三张无规则效果牌负责食性与牌面教学，其余牌逐步展示甜点留存和兔子规模。
- 基础选择仍是吃与弃：吃可食用牌获得牌面分及每张实体牌每轮首次的 1 金币；错误食性通常为负分。
- 【后置】将当前牌移动到本次餐盘最后，不执行当前牌的吃弃效果，也不计行动；每张实体牌每轮限一次。后置联动牌可以改变这次移动或为它追加有限收益。
- 餐盘初始容量 10。大牌组不会自动增加每轮行动，必须在商店投资永久扩容。
- 重洗次数来自卡牌或道具；牌堆清空后自动、无缝触发，可叠加，不再需要点击重洗按钮。
- 未变化的吃/弃点数统一显示为白色；高于原值才显示绿色，低于原值才显示红色，并同时显示相对原值的差额。
- 【净化】只恢复低于原值的红色点数，绿色成长保留；【弱化】生成牌在吃或弃结算后都会摧毁自身。
- 【硬吃】故意选择错误食性，承受较低牌面并换取额外分数、连击、蓄势和合约金币，形成与“永远按正确食性”相反的构筑路线。
- Fable5 扩展牌加入后置成长、厌食转化、跨轮储存、牌堆标记、目标延缓与商店资源；每种效果都使用独立规则维度，而不是只改倍率。

### 八个流派

- 水果：基础苹果用于教学，其余水果围绕【水果连击】；后置可把中间的非水果移开，部分引擎生成【弱化】水果维持循环。
- 快餐：普通快餐吃点不超过 +2；超过 +2 的牌必须提高稀有度并附带【厌食】、金币成本或自我摧毁等代价。
- 甜点：弃掉触发【留存】，永久积累吃分；达到阈值后吃下会翻倍或三倍爆发并重置。
- 饮料：多数在触发后摧毁自身，换取自动重洗、指定类别的延迟增益或【弱化】生成牌；押金瓶则用负分换即时金币。
- 动物：橘猫是基础牌，其余动物可【摧毁】上一张匹配牌、生成牌或降低别的牌来成长。兔子继续按永久牌组中的兔子数量计分。
- 星体：直接改变机制，包括交换剩余牌面、增加自动重洗、冻结计时、清场、随机生成与后置兑现。
- 人物：连接其他类别；债务经纪人以负分换金币，天文学家与魔术师生成会自行离场的【弱化】牌。
- 通用：负责净化、点数转移、硬吃与商店折扣；榨分机能把水果的红色降点转成自身绿色成长。

## 合约、经济与商店

- 同时只持有一条持续合约：未完成会跨轮保留，完成后奖励金币、标记完成并移除；下一轮再从三条当前牌组可完成的合约中选择一条。合约不增加牌面或最终倍率。
- 永久限时经济：12 秒内完成本轮额外 +1 金币，8 秒内再 +1。
- 普通模式不出现危险任务；首次通关后解锁高难模式，高难模式在第 4 / 8 / 12 轮强制进行危险任务三选一。代价立即生效，要求只追踪当轮，完成后下一轮获得一件不能在普通商店购买的高级道具。
- 商店同时出售三张随机卡牌和一个三张同类别的专柜，既保留发现感，也允许定向补强流派。
- 普通 / 罕见 / 稀有 / 传奇牌基准价提高为 5 / 8 / 12 / 18；基础白板和有明显代价的牌会单独降价。
- 商店刷新按 2、3、4……金币递增；免费刷新仍推进后续价格。打折券每轮只提供 -1 卡价。
- 每间商店只展示 2 件道具。活跃道具池扩为 24 件，弱效果道具更便宜，自动重洗和永久折扣是中后期高价投资；纸果篮每轮提供一张【弱化】苹果。
- 经济类道具已压低前期购入价，使其能在 15 轮内回本；高难任务奖励仍明显强于普通商店道具。
- 商店可锁定；锁定后下一轮保留未购买的随机卡、同类专柜卡与道具，不会重新抽取。
- 删牌按 0、3、6……金币递增且没有返还；删除前必须二次确认。所有模式每完成 5 轮额外累积 1 次免费删牌。
- 阶段累计目标为第 5 / 10 / 15 轮的 100 / 300 / 500 分。

## 解锁模式与游戏菜单

- 本机浏览器只要保存过一次通关记录，欢迎页就会永久显示“无尽模式”和“高难模式”入口。
- 无尽模式的前 15 轮与普通模式完全相同；通过第 15 轮 500 分门槛后进入第 16 轮，此后不再设置累计分数门槛。
- 高难模式保留普通模式的 15 轮目标，并增加第 4 / 8 / 12 轮的必选危险任务。
- 右上角菜单可以分别开关音乐与音效，在小 / 标准 / 大三档界面字号之间切换，并打开包含全部 89 张牌的卡牌图鉴。
- 音乐、音效与字号设置写入浏览器记录；字号只调整说明文字与操作控件，不改变卡牌点数、牌堆几何或触摸区域。

## 工程边界

- `js/data.js`：当前 89 张、可继续扩展的卡池和七张不同教学卡组成的初始牌组。
- `js/keywords.js`：关键字词典、效果映射与“摧毁”术语归一化。
- `js/rules.js`：26 条单轮金币合约。
- `js/items.js`：12 件高难任务高级道具、24 件活跃商店道具及其结算。
- `js/quests.js`：高难模式第 4 / 8 / 12 轮的任务草案、惩罚、判定与延迟奖励。
- `js/plate.js`：餐盘抽取、扩容与后置状态转换。
- `js/reshuffle.js`：可叠加重洗状态与纯规则结算。
- `js/engine.js`：不依赖 DOM 的卡牌、合约、计分和经济核心。
- `js/gesture.js`：纵向吃弃与横向后置的 Pointer Events 输入。
- `js/shop.js`：随机货架、同类专柜、道具、刷新、扩容与删牌。
- `js/state.js`：状态机与存档形状。
- `js/ui.js` / `js/main.js`：DOM 渲染与流程编排。
- `test/core.test.js`：卡池、白板教学、弱化、净化、降点转移、后置、重洗、合约、商店及 UI 契约测试。
- `scripts/browser-smoke.mjs`：Microsoft Edge 的手机/桌面四轮冒烟测试，覆盖模式解锁、图鉴、大字号、商店锁定和高难任务。
- `docs/CARD_ART_SPRITESHEET_PROMPTS_V0.17.md`：全部 89 张卡的 5 张 4×4 + 1 张 3×3 统一风格生图提示词与切图文件名。

Cloudflare 步骤见 [`docs/CLOUDFLARE_DEPLOY.md`](./docs/CLOUDFLARE_DEPLOY.md)。
