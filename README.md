# 吃牌肉鸽 · CardEater

一个无依赖、移动端优先的原生 H5 构筑肉鸽 Demo。向下吃、向上弃；110 张卡牌通过首末位、前后相邻、类别、牺牲、生成、摧毁、重洗、永久成长与经济效果形成构筑，74 条不重复规则会在一局中永久叠加。

## 本地运行

ES Modules 需要通过 HTTP 打开，不能直接双击 `index.html`：

```powershell
npm start
```

然后访问 `http://localhost:8080`。常用验证命令：

```powershell
npm test
npm run check
npm run build
```

`npm run build` 会生成可直接部署到 Cloudflare Pages 的 `dist/`。

## 当前玩法

- 初始 7 张牌保持 4 张可食用、3 张不可食用；每张初始牌都带轻量位置效果，让第一轮就开始观察首位、末位和相邻牌。
- 所有复杂效果统一使用【摧毁】【相邻】【成长】【生成】【重洗】【净化】【追溯】【蓄势】【位置】【规模】【复制】【经济】【储存】【预判】等关键字；成长牌会直接写出固定数值或完整 `+X` 公式。
- 右上角“组”按钮在游玩、规则选择、结算和商店中都可打开永久牌组，查看每张实体牌当前吃/弃分、成长进度、储存分、完整效果与关键字释义。
- “组 / 任 / 声音”已归入固定顶栏；普通弹层从顶栏下方开始，牌组详情只让卡牌列表滚动，避免入口、关闭按钮和说明区互相遮挡。
- 中后期通过商店构筑水果、快餐、甜点、饮料、蔬菜、星体、人物、动物和通用九类牌；任务还会加入吃弃皆为 -1、无效果、无类别的虚空牌。
- “可食用”不等于永远正分：腐烂苹果、黑咖啡、能量饮料等需要先承受负分，再把收益兑现到后续牌。
- 每轮规则三选一，选中后永久保留且不会重复；规则已覆盖纯弃牌、低吃牌、大/小牌组、重洗、重复触发、类别、首末位和交替节奏。
- 第 3、6、9、12 轮追加危险任务三选一：接受当轮或永久负面；要求、惩罚和奖励可随时从右上角任务按钮查看。达标后会在结算栏明确标记“任务完成，下一轮生效”。
- 小牌组可通过“重启按钮”或“袖珍洗牌机”重洗已经操作过的牌，重复触发永久成长和节奏引擎。
- 每轮餐盘默认最多出现 10 张牌；20 / 30 / 40 / 50 张牌组才分别提高到 11 / 12 / 13 / 14 张，未登场的牌留在备选区。大牌组因此提供覆盖面与稳定性，不再按总张数线性增加行动。
- 8 张以内获得 ×1.15 构筑密度倍率，9–10 张为 ×1.07；基础金币只计算前 6 次吃牌，13 张起每超载 4 张收取 1 金币携带费。
- 商店购买后不会免费补货；刷新从 3 金币起，每次增加 2 金币，免费刷新后仍可正常购买并会给出准确失败原因。13 张起的牌组扩张会逐阶提高卡牌售价，最高 +5；所有卡牌都有明确持有上限，商店另售 14 件本局唯一低级道具。
- 回收卡牌按 0、5、10……递增收费，并按稀有度返还 0–5 金币；超载牌组再获得最多 +3 回收奖励。扩张提高后续成本，删牌则同时恢复抽中率、金币和商店价格。
- 阶段累计目标为第 5 轮 150、第 10 轮 1500、第 15 轮 12000。
- 分数、倍率、金币和音高都经过有限值与上限保护，长连击不会再产生 `Infinity` / `NaN`。

## 代码边界

- `js/config.js`：15 轮、阶段目标等平衡参数。
- `js/balance.js`：稀有度价格、商店权重和价值预算。
- `js/data.js` / `js/rules.js`：可序列化内容数据。
- `js/keywords.js`：统一关键字词典、效果到关键字的映射与“摧毁”术语归一化。
- `js/deck-pressure.js`：餐盘预算、构筑密度、金币饱腹、携带费、扩张附加价与超载回收奖励。
- `js/quests.js` / `js/items.js`：任务风险、达成条件与永久道具。
- `js/numbers.js`：跨平台可复刻的有限值、饱和运算与紧凑数字显示。
- `js/state.js`：严格状态机和存档形状。
- `js/engine.js`：不依赖 DOM 的行为、效果和计分核心。
- `js/gesture.js`：浏览器 Pointer Events 输入层。
- `js/platform.js`：时间、随机数、ID、震动、本地记录适配层。
- `js/ui.js`：DOM 渲染；`js/main.js`：流程编排。
- `assets/cards/*.webp`、`cards-atlas.webp` 与 `meta-atlas.webp`：导出器按 alpha 主体自动去除跨格碎片、缩放到统一安全区并居中；H5 开局预载 7 张独立卡图，游玩期间加载中后期单图集，任务/道具共用元图集。新增 50 张卡的 5 组 5×2 源精灵图保存在 `assets/card-sprites-set-7.png` 到 `set-11.png`。
