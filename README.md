# 吃牌肉鸽 · CardEater

一个无依赖、移动端优先的原生 H5 构筑肉鸽 Demo。向下吃、向上弃；110 张卡牌通过首末位、前后相邻、类别、牺牲、生成、摧毁、重洗、永久成长与经济效果形成构筑，70 条分阶段解锁且不重复的规则会在一局中永久叠加。

## 本地运行

ES Modules 需要通过 HTTP 打开，不能直接双击 `index.html`：

```powershell
npm start
```

然后访问 `http://localhost:8080`。常用验证命令：

```powershell
npm test
npm run check
npm run build
```

`npm run build` 会生成可直接部署到 Cloudflare Pages 的 `dist/`。

## 当前玩法

- 初始 7 张牌保持 4 张可食用、3 张不可食用；每张初始牌都带轻量位置效果，让第一轮就开始观察首位、末位和相邻牌。
- 所有复杂效果统一使用【摧毁】【相邻】【成长】【生成】【重洗】【净化】【追溯】【蓄势】【位置】【规模】【复制】【经济】【储存】【预判】等关键字；成长牌会直接写出固定数值或完整 `+X` 公式。
- 右上角“组”按钮在游玩、规则选择、结算和商店中都可打开永久牌组，查看每张实体牌当前吃/弃分、成长进度、储存分、完整效果与关键字释义。
- “组 / 规 / 具 / 任 / 声音”已归入固定顶栏；“规”和“具”可随时查看本局已经选择的永久规则与已经获得的道具。
- 中后期通过商店构筑水果、快餐、甜点、饮料、蔬菜、星体、人物、动物和通用九类牌；任务还会加入吃 -2、弃 -1、无效果、无类别的虚空牌。
- “可食用”不等于永远正分：腐烂苹果、黑咖啡、能量饮料等需要先承受负分，再把收益兑现到后续牌。
- 开局页直接写明第 5 / 10 / 15 轮的累计目标和每轮流程；规则选择页持续显示当前分、还差多少、剩余轮数和进度条。
- 每轮规则三选一，选中后永久保留且不会重复；容易完成的高倍率规则延后至第 2～4 轮，卡片会标注基础/进阶/后期与解锁轮次。
- 规则池不再包含任何要求重洗、重复处理同一张牌或重洗后计分的倍率规则；获得重洗只增加行动、效果与成长机会。
- 第 3、6、9、12 轮追加危险任务三选一：接受当轮或永久负面；要求、惩罚和奖励可随时从右上角任务按钮查看。达标后会在结算栏明确标记“任务完成，下一轮生效”。
- 小牌组可通过“重启按钮”“回卷发条”“秒表”或“袖珍洗牌机”重洗已经处理过的牌；次数可叠加，空牌堆时会明确选择重洗或结束本轮。再次处理仍可触发效果和成长，但同一张实体牌每轮只产 1 次基础金币。
- 永久餐盘初始容量为 10：每轮只从完整牌组随机抽取至多 10 张登场，其余牌留作下轮候选。商店可以付费永久扩容，每次 +1。
- 餐盘扩容价格按 2、3、5、8、12、17……递增。大牌组玩家需要在买牌提升卡池与扩容提高登场数之间分配金币，而不是无条件获得更多行动。
- 牌组大小不再自带倍率、金币上限、携带费或卡价惩罚；基础金币等于本轮被吃过的不同实体牌数，重洗不重复印钱。
- 商店购买后不会免费补货；刷新按 1、2、3……递增，免费刷新仍会推进刷新次数。普通/罕见/稀有/传奇牌保持 3/6/10/16，使前期操作便宜、后期核心投资昂贵。
- 收银台弃掉后摧毁自身，并让随后商店获得 2 次免费刷新，使其收益与 3 金币买价及 1/2 金币刷新曲线匹配。
- 商店道具池为 24 件且各有独立居中像素图。经济投资件标注最早与最晚出现轮次，后期不会再抽到无法回本的金币、刷新、扩容或折扣道具；回卷发条仍为 16 金币、第 7～13 轮出现。
- 删牌费用按 0、3、6……递增，不返还金币；点击牌后必须在浮窗再次确认，删牌的收益只来自提高目标牌登场概率。
- 阶段累计目标为第 5 轮 120、第 10 轮 750、第 15 轮 4000；任务基准为 25 / 90 / 300 / 1000。
- `npm run simulate` 会让无作弊的策略代理实际经历规则、任务、吃弃、重洗、商店买牌、扩容、删牌和结算；默认同时输出一局 15 轮日志与三类构筑的 40 种子通关率。
- 分数、倍率、金币和音高都经过有限值与上限保护，长连击不会再产生 `Infinity` / `NaN`。

## 代码边界

- `js/config.js`：15 轮、阶段目标等平衡参数。
- `js/balance.js`：稀有度价格、商店权重和价值预算。
- `js/data.js` / `js/rules.js`：可序列化内容数据。
- `js/keywords.js`：统一关键字词典、效果到关键字的映射与“摧毁”术语归一化。
- `js/plate.js`：永久餐盘容量、每轮登场抽样与扩容价格曲线。
- `js/quests.js` / `js/items.js`：任务风险、达成条件与永久道具。
- `js/numbers.js`：跨平台可复刻的有限值、饱和运算与紧凑数字显示。
- `js/state.js`：严格状态机和存档形状。
- `js/engine.js`：不依赖 DOM 的行为、效果和计分核心。
- `js/gesture.js`：浏览器 Pointer Events 输入层。
- `js/platform.js`：时间、随机数、ID、震动、本地记录适配层。
- `js/ui.js`：DOM 渲染；`js/main.js`：流程编排。
- `assets/cards/*.webp`、`cards-atlas.webp`、`meta-atlas.webp` 与 `shop-items-atlas-v013.webp`：导出器按 alpha 主体自动去除跨格碎片、缩放到统一安全区并居中；H5 开局预载 7 张独立卡图，游玩期间加载中后期单图集，24 件商店道具使用独立 6×4 图集。新增 50 张卡的 5 组 5×2 源精灵图保存在 `assets/card-sprites-set-7.png` 到 `set-11.png`。
